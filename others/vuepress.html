# 构建自己的 Vuepress 项目

要将页面托管到 GitHub Pages，首先要进行构建

在项目根目录运行 npm run docs:build 进行构建，成功后会在 docs/.vuepress/ 下生成 dist 文件夹

这个文件夹下的内容就是要上传到 GitHub 仓库的内容

TIP

指令不一定是 npm run docs:build，查看 package.json，在 scripts 中确定具体要运行的指令

一般来说，scripts 中的内容应该大致长这样

"scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
}

找到 "vuepress build docs" 对应的 key（此处为 docs:build），然后运行 npm run <key>

WARNING

如果 Node 的版本过高（>=17），很可能在构建过程中会报错
error:0308010C:digital envelope routines::unsupported

不同命令行下有不同的解决办法，在 powershell 中，先执行一次
$env:NODE_OPTIONS="--openssl-legacy-provider"
然后重新执行 npm run docs:build 就可以正常构建

# 在 GitHub 中建立仓库

在 GitHub 中新建一个名为 <USERNAME>.github.io 的仓库，其中 <USERNAME> 是 GitHub 的用户名

创建完成后，点击右上角的 Settings，在左侧 Code and automation 中找到 Pages

进入 GitHub Pages 配置页面，在 Branch 栏中修改 None 为 master，点击 save

至此仓库的配置就完成了

# 上传页面至仓库

进入 docs/.vuepress/dist/ 依次执行以下指令

git init
git add -A
git commit -m 'deploy'

git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master

如果先前在本机上配置过 ssl，那么 dist 下的文件就应该成功上传到仓库中了

访问 https://<USERNAME>.github.io 即可看到自己的页面（如果用户名中有大写字母，需要转为小写）

# 附

用于一键上传的 powershell 代码，在项目根目录运行即可

$env:NODE_OPTIONS="--openssl-legacy-provider"

npm run docs:build

cd docs/.vuepress/dist

git init
git add -A
git commit -m 'deploy'

git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master
