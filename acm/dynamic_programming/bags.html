# 背包问题

简单地说，背包问题就是一堆板子，请背诵圣经

# 0-1 背包

有一个至多可以承载重量为 W 的背包，还有 n 个物体。忽略物体实际几何形状，每个物体仅有两个属性——重量 weight 和价值 value

问：背包可以获得的最大价值是多少？

时间复杂度 

vector<int> dp(W + 1, 0);
for (int i = 1; i <= n; i++) {  // 枚举第 i 个物品
    for (int j = W; j >= weight[i]; j--) {
        dp[j] = max(dp[j], dp[j - weight[i]] + value[i]);
    }
}

# 完全背包

与 0-1 背包不同的是，完全背包中的每个物品可以取无数次，而 0-1 背包中每个物品只能取 0 / 1 次。

时间复杂度 

for (int i = 1; i <= n; i++) {  // 枚举第 i 个物品
    for (int j = 0; j <= W - weight[i]; j++) {
        dp[j + weight[i]] = max(dp[j + weight[i]], dp[j] + value[i]);
    }
}

# 多重背包

多重背包也是 0-1 背包的一个变式。与 0-1 背包的区别在于每种物品有 s 个，而非一个

一个很朴素的想法是：把「每种物品最多选 s 次」转换为「有 s 个相同的物品，每个物品选一次」。这样就转换成了一个 0-1 背包问题

时间复杂度 

for (int i = 1; i <= n; i++) {  // 枚举第 i 个物品
    for (int j = W; j >= weight[i]; j--) {
        for (int s = 1; s <= num[i] and (j - s * weight[i]) >= 0; s++) {
            dp[j] = max(dp[j], dp[j - s * weight[i]] + s * value[i]);
        }
    }
}

# 二进制分组优化

朴素算法在 s 很大时会超时，往往采用二进制优化，时间复杂度 

vector<pair<int, int>> goods;
for (int i = 1; i <= n; i++) {
    int v, w, s; cin >> v >> w >> s; 
    for (int k = 1; k <= s; k *= 2) {
        goods.emplace_back(k*v, k*w);
        s -= k;  // 二进制分组
    }
    if (s > 0) goods.emplace_back(s*v, s*w); // n - 2k + 1
}
