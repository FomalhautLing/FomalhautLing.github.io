# 并查集

# 简介

并查集（Disjoint Set Union. DSU）是一片森林。森林中的每一棵树表示一个集合

理想的并查集，它的每一棵树都是一个“菊花”，即中间一个根节点，外边一圈都是指向根的子节点。这种形态的并查集有最高的查询效率，因为一般的树在寻找根节点时，还有一个“爬树”的过程

并查集的初态，所有的节点都指向自己，表示自己就是自己的根节点

# 板子

一般也不需要用到这样一个结构体。并查集本质上就是一个 fa 数组

struct DSU {
private:
    vector<int> fa;
    int n;

public:
    explicit DSU(int n): n(n) {
        fa.resize(n + 1);
        for (int i = 0; i <= n; i++) fa[i] = i;
    }

    int find(int x) {
        return x == fa[x] ? x : find(fa[x]);
    }

    void merge(int x, int y) {
        fa[find(x)] = find(y);
    }
};

# 作用

# 判断两个元素是否属于同一个集合

将 n 个数划分成 m 个集合，然后多个询问，询问给出 x y，回答 x 和 y 是否在同一个集合中

这是并查集的本职工作，它就是干这个的

例题：P2256 (opens new window)、P3367 (opens new window)

# 按大小合并 / 按秩合并

基础的并查集合并没有规律，现在赋予它规律，使得它可以额外维护一些信息

# 按大小合并

需要一个额外的 sz 数组，记录每个集合（每棵树）中节点的个数
将节点少的集合合并到节点多的集合中，这样查询时会更快
sz 数组应当初始化为 1，表示集合中只有一个元素，就是个根节点本身

void merge_size(int x, int y) {
    x = find(x), y = find(y);
    if (sz[x] > sz[y]) swap(x, y);
    fa[x] = y;
    sz[y] += sz[x];
}

# 按秩（树高）合并

需要一个额外的 rnk 数组，记录树高
将树高较小的合并到较大的中，这样查询时会更快
rnk 数组可以初始化为 0 或 1，这取决于对树高的定义是叶子到根的边数还是节点数

void merge_rank(int x, int y) {
    x = find(x), y = find(y);
    if (rnk[x] > rnk[y]) swap(x, y);
    fa[x] = y;
    if (rnk[x] == rnk[y]) rnk[y]++;
}

# 带权并查集

带权并查集维护节点到根的距离

int find_weighted(int x) {
    if (x != fa[x]) {
        int rt = find(fa[x]);
        value[x] += value[fa[x]];
        fa[x] = rt;
    }
    return fa[x];
}

void merge_weighted(int x, int y, int w) {  // x 到 y 的距离是 w
    int rx = find(x), ry = find(y);
    fa[rx] = ry;
    value[rx] = value[y] - value[x] + w;
}
