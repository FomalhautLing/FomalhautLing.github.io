# 正则

正则是一套规则，用于对字符串进行模式匹配

学习工具 (opens new window)

# 基本匹配

# abc

匹配 "abc"

# [abc]

匹配 "a" 或 "b" 或 "c"，整个方括号被视为一个字符

# [^abc]

匹配 "a"、"b"、"c" 以外的所有字符，整个方括号被视为一个字符

# [a-z]

匹配 a-z 中的任意一个字符。它是一种简写，除此以外还有 [A-Z] 以及 [0-9]
它们可以写在一起 [a-zA-Z0-9]

# a|b

匹配 "a" 或 "b"

# 点号 .

匹配除 \n \r 以外的任意一个字符，相当于 [^\n\r]

DETAILS

Vuepress 会给标题生成锚点链接，但锚点中不允许点号 . 出现
本标题的 url 就不是 ../regex.html#点号.

# 限定符

  1. +：出现一次或多次
  2. *：出现零次或多次
  3. ?：出现零次或一次
  4. {n}：出现 n 次
  5. {n, }：出现 n 次及以上
  6. {n, m}：出现 n 到 m 次，包含端点

# 转义匹配

# \s 与 \S

\s：匹配空白字符，包括空格、制表符、换页符等等，相当于 [ \f\n\r\t\v]

\S：匹配非空白字符，相当于 [^\s]

# \d 与 \D

\d：匹配数字，相当于 [0-9]

\D：匹配非数字，相当于 [^\d]

# \w 与 \W

\w：匹配手写字符，相当于 [a-zA-Z0-9_]

\W：匹配非手写字符，相当于 [^\w]

# 分组匹配

圆括号内的字符被视为一个组，一般与或连用

# 捕获组

圆括号圈起来的，例如 Phone Number: ([0-9]{11})

# 非捕获组

圆括号圈起来，且有 ?: 前导，例如 My Like: ((?:apple)|(?:banana))

# 断言组

断言组也是非捕获组

# abc(?=\d+)

匹配 "abc"，且后面紧跟着一串数字

# (?<=\d+)abc

匹配 "abc"，且前面紧跟着一串数字

# abc(?!\d+)

匹配 "abc"，且后面不紧跟着一串数字

# (?<!\d+)abc

匹配 "abc"，且前面不紧跟着一串数字
