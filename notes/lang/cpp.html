# 复数

Cpp 中有三种复数类型，分别是 _Complex, __complex__ 和 complex<T>，其中前两种是为兼容 C 语言而提供的，这里介绍的是最后一种

复数不支持隐式类型转换，两个类型不同的复数（例如 complex<int> 与 complex<double>） 无法计算

# 虚数单位字面量

Cpp 的虚数单位字面量是小写的 i，它被默认为 complex<double> 类型

TIP

除此以外，I, j, J 都是合法的虚数单位字面量，但它们被认为是 _Complex int 类型

li, lI, lj, lJ 等都被认为是合法的后缀，但它们都会被解释成 _Complex long

由于虚数单位默认是 complex<double> 的，所以这个写法无法通过编译 auto a = 1 + 2i;

# regex_match(s, m, re)

对文本串 s，以 re 为正则模板进行一次全文匹配，匹配结果放在 m 中。返回 true 如果匹配成功，否则返回 false

该函数重载了多次，m 也可以缺省，只用于判断 re 是否可以匹配 s

# regex_search(s, m, re)

对文本串进行一次全文查找，它的参数与 regex_match 一样，返回值也一样

匹配与查找的区别：匹配是指文本串全文可以被 re 匹配，查找是指文本串中有一部分可以被 re 匹配

当 search 的结果存在 regex_match 中时，可以使用 .prefix() 和 .suffix() 方法查看匹配成功的部分的前文和后文

# regex_replace(s, e, fmt)

对文本串 s 进行全文替换，将匹配正则 e 的部分替换为 fmt

这里 fmt 是一个字符串，里面可以使用 $ + num 的形式引用正则中的分组

string text = "python.cpp";
smatch result;  // 匹配结果
regex pattern = regex(R"(([a-z]+)\.(c|cpp|py))");  // 正则

if (regex_match(text, result, pattern)) {
    cout << result[0] << endl;  // 第 0 项是整个字符串本身
    cout << result[1] << endl;
    cout << result[2] << endl;
}

>>> python.cpp

# Cpp 17

# string_view

string_view 封装了 string（指针） 和 它的长度，并提供了 string 的所有只读接口

string_view 不实际拥有字符串，它的拷贝是常数级的，它只需要拷贝指针和长度

与 string& 的区别：string_view 是视图，它进行的操作实际上只是指针在进行移动，而在 string& 上进行操作时，例如 sub_str，实际上会生成一个新的 string

WARNING

string_view.data() 和 string_view 本身的打印值很大可能是不一样的。string_view 很可能不包含 '\0'，这会导致打印时缺少字符串边界

# ranges::

ranges 库是对 STL 的扩展

STL 使用迭代器 begin(), end()，标志一个容器的范围，algorithm 库也依此设计，这在某些场景下相当冗余（例如 std::sort(vec.begin(), vec.end())），ranges 应运而生

ranges 实际上是对 begin(), end() 的封装。ranges 库提供的函数都适用于 STL

# sort

ranges::sort(iterable) 相当于 std::sort(iterable.begin(), iterable.end())

vector<int> arr{2, 0, 2, 4};
ranges::sort(arr);
>>> [0, 2, 2, 4]

ranges::sort(arr, greater<>());  // 也可以自定义比较方式
>>> [4, 2, 2, 0]

# views::

视图，它是 ranges 的一种，但它只引用元素，并且无论包含多少个元素，都可以在固定时间内复制、分配和销毁

视图是惰性求值的，不访问就不占时间

# 管道运算符

views 库中的函数都接受一个 Range 对象作为自己的第一个参数，实际在处理数据时，会导致函数的深层嵌套，为此出现了管道运算符

vector<int> arr{2, 0, 2, 4};
views::take(views::transform(arr, [](int x) { return x * x; }), 2);

// 与下面的写法等价
arr | views::transform([](int x) { return x * x; }) | views::take(2)

下面这些仿函数的类型为 RangeAdaptor，称为范围适配器，它们可以对 range 或 view 进行特定的处理

# filter

filter([](int x) { return x & 1; })，不满足条件的被过滤

# transform

transform([](int x) { return x * x; })，对数据做逻辑处理

# take / take_while

take(10)，取前 10 个，若已经超出范围，自动截断

take_while([](int x) { return x & 1; })，取直到遇到第一个偶数

# drop / drop_while

drop(5)，舍弃掉前 5 个
drop_while([](var x) { return x & 1; })，舍直到遇到第一个偶数

# reverse

reverse，反转区间，不要加括号

# elements / keys/ values

elements<1>，只取 tuple 中下标为 1 的元素

keys，等价于 elements<0>

values，等价于 elements<1>

# split

spilt(x)，用元素 x 对 view 进行分割，这会返回一个嵌套的 view

# join

join，合并一个嵌套的 view

# iota

iota(a, b)，范围整数区间 [a, b) 中的所有数，这个生成也是惰性的，有访问才生成
