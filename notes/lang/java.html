# Lambda

对于只有一个方法的接口，可以使用 lambda 表达式替换

lambda 并不是简单的语法糖，匿名内部类是会被编译出 class 文件的，而 lambda 不会

lambda 实际上会在当前类中生成一个新的方法，其异常也会在该方法中抛出

lambda 只支持仅有一个抽象方法的接口，不支持抽象类

Thread thread = new Thread(new Runable() {  // 以前需要使用匿名内部类
    @Override
    public void run() {
        System.out.println("Hello World");
    }
}).start();

Thread thread = new Thread(() -> {  // java8 以后
        System.out.println("Hello World");
    }
}).start();

Runable runable = () -> {};  // 甚至可以这么写

# .of(T value)

将 value 包装进去，此处 value 不得为 null，否则抛异常

# .ofNullable(T value)

将一个可空的 value 包装进去

# .ifPresent(Consumer consumer)

如果非空则执行 consumer

# .get()

获取包装进去的 value，value 不得为空，否则抛异常

# .orElse(T other)

功能同 get，当 value 为空时，返回 other

# .stream()

集合类都有 .stream() 方法，用于获取它们的流

对于数组，可以使用 Array.stream() 或者 Stream.of()

# .skip(long n)

跳过前 n 条数据，如果流的长度小于 n，则得到一个空流

# .limit(long maxSize)

限制通过的数据个数。如果 maxSize 比流的大小要大，则没有任何影响

# .sorted()

排序

# .distinct()

去重

# .filter(Predicate predicate)

不满足条件的被过滤

# .map(Function mapper)

对流中的数据做映射，一般用于类型转换

# .count()

这是一个终端方法，返回流的大小

# .forEach(Consumer action)

这是一个终端方法，对流中的数据做最后的处理

# .collect(Collector collector)

将流转为集合。例如 .collect(Collectors.toList()) 返回一个 list

# Java 9

# 模块机制

模块是包的集合

随着依赖数量的增加，由于默认所有 jar 包中的类都可以被访问，这对 IDE 建立索引产生了很大负担，同时建立出来的大量候选类中往往只有一小部分会被使用，这时候就需要主动地屏蔽掉一些冗余的类

模块对包提供了一个可见性的控制，它允许包的制作者主动暴露（export）出可以被使用的包，而包的使用者可以主动依赖（require）暴露出来的包

# 模块的引入

在 src 目录下创建 module-info.java 文件，表示此项目采用模块管理

module exampleModule {  // 本项目的模块名
    requires java.logging;  // 依赖 java 日志模块
    exports com.example;  // 暴露本项目中的包出去
    exports com.example to anotherModule;  // 暴露给特定的模块
    requires transitive lombok;  // 默认情况下依赖不会传递，需要显式指明
}

        yield "不及格";  // 在逻辑块中使用 yield 来返回
    }
};

# Sealed 类型

final 修饰的类不可被继承，而 sealed 打开了一个口子，它允许其修饰的类被指定的类继承

public sealed class A permits B, C { ... }  // A 向 B，C 开放继承

public non-sealed class B extends A { ... }  // B 继承 A
1
2
3

警告

继承 sealed 类的子类必须显式地指明为 final / sealed / non-sealed（重新开放继承） 其中之一
